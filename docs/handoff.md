# Protoboard — руководство для нового чата (передача состояния)

Этот файл — краткая, но полная карта проекта: что это, как устроено, что готово,
что в работе и на что обратить внимание. Прочитай его первым в новом чате.

> Пользователь — не сильный технически. Объяснять простыми словами, идти по шагам,

> «брифу» и поэтапно.

## 1. Что это

**Protoboard** — веб-трекер задач и багов для команды (~15–20 чел.) по прототипам
мобильных игр. Упрощённая Asana: один проект = один прототип. Доступ по ссылке,
изменения видны всем в реальном времени.

## 2. Стек и инфраструктура

- **Фронтенд:** React + Vite (одностраничное приложение). Сборка `npm run build`.
- **База данных:** **Firebase Firestore** (заменил Supabase в июне 2026).
  - Проект: `protoboard-fb` (firestore region: `eur3`).
  - Ключи — в `.env.local` (локально) и `.env.production` (для CI/CD, закоммичено).
    apiKey Firebase публичный по задумке — защита через Firestore Security Rules.
  - Rules: `allow read, write: if true;` (без авторизации, открыто всем).
  - Коллекции: `projects`, `tasks`, `attachments`.
- **Хостинг:** **GitHub Pages** → https://calm4.github.io/protoboard/
  - Репозиторий: `github.com/Calm4/protoboard` (личный аккаунт пользователя).
  - Авто-деплой: GitHub Actions (`.github/workflows/deploy.yml`) на каждый push в `main`.
  - **НЕ Cloudflare Pages** — `*.pages.dev` заблокирован у пользователя в регионе.
- **Скриншоты:** хранятся как **base64 data URL прямо в Firestore** (коллекция `attachments`).
  Firebase Storage не используется (требует платный Blaze-план). Cloudinary также заблокирован
  в регионе пользователя. Лимит Firestore на документ — 1 МБ; сжатие до ≤1920px JPEG
  держит большинство скриншотов в пределах 800 КБ base64.
- **Деплой делает пользователь:** через **GitHub Desktop → Push**. Агент пушить НЕ может.
  Коммитить агент может; пушит — пользователь.

## 3. Структура кода

```
src/
  App.jsx                 — главный компонент: состояние UI, открытие задачи по ссылке,
                            экраны загрузки/ошибки, сборка экранов
  hooks/
    useProjects.js        — ВСЯ работа с данными: Firestore onSnapshot, CRUD
                            проектов/задач/статусов, скриншоты (lazy load + upload),
                            кэш скриншотов (shotsCacheRef), отмена действий
    useAuth.js            — задел под вход (AUTH_REQUIRED=false, вход выключен)
  lib/
    firebase.js           — клиент Firebase (читает .env VITE_FIREBASE_*)
    image.js              — сжатие картинок (canvas → JPEG) + скачивание
    color.js              — hexToRgba (тинты для плашек)
  components/
    ProjectGrid.jsx       — главная: список проектов + архив
    ProjectView.jsx       — экран проекта: шапка, поиск, фильтр платформы, доска/список
    Board.jsx             — доска: колонки=статусы, dnd карточек и колонок, верх. скролбар
    TaskList.jsx          — список: группы по статусам, инлайн-выпадашки, dnd
    TaskPanel.jsx         — панель задачи: поля, скриншоты, #N + «скопировать ссылку»
    StatusMenu.jsx        — меню статуса (переименовать/цвет/удалить)
    Editable.jsx          — EditableInput/EditableTextarea (правка по Enter/blur, авто-рост)
    ColorSwatches.jsx     — палитра цветов (проект/статус)
    NewProjectModal.jsx   — окно создания проекта (имя + цвет)
    LoginScreen.jsx       — экран входа (показывается только если AUTH_REQUIRED=true)
    DeleteProjectModal.jsx — модал удаления проекта (вводить «DELETE»)
    StatsView.jsx          — статистика: карточки, диаграммы, спарклайн
  constants.js            — PRIORITIES, PLATFORMS, DEFAULT_STATUSES, PROJECT_COLORS,
                            GRADIENTS (12 пресетов), EMPTY_FILTERS (showClosed:false)
  styles.js               — все стили (строка CSS, классы .pb-*)
scripts/
  migrate-to-firebase.js  — одноразовый скрипт миграции Supabase→Firebase (уже выполнен)
docs/                     — этот файл
```

**Ключевая идея архитектуры:** все данные идут через `useProjects.js`. Компоненты
получают данные и колбэки оттуда.

**Поля задачи в приложении:** `title`, `notes`, `priority` (high/med/low), `status` (id из
статусов проекта), `platform` (ios/android/both), `version`, `order` (дробный, sortOrder в
Firestore), `num` (#N), `tags[]`, `shots[]` (lazy), `shotsLoaded` (bool),
`activity[]` (lazy), `activityLoaded` (bool), `dueDate` (YYYY-MM-DD), `assignee` (строка),
`completedAt` (timestamp ms | null — ставится при попадании в последний статус),
`closed` (bool — «выполнено» вручную, независимо от статуса).

**Поля задачи в Firestore:** `projectId`, `title`, `description`, `notes`, `priority`, `status`,
`platform`, `version`, `sortOrder`, `num`, `tags[]`, `createdAt`, `dueDate`, `assignee`,
`completedAt`, `closed`.

**Проект:** `name`, `build`, `archived`, `color`, `statuses[]` (`{id,label,color}`),
`customTags[]`, `gradient` (CSS gradient string | "").

**Activity (коллекция `activity`):** `projectId`, `taskId`, `action` (string), `timestamp` (ms).
Лог анонимный (без пользователей). Загружается лениво при открытии задачи.

**Attachment (коллекция `attachments`):** `taskId`, `name`, `data` (base64 data URL), `createdAt`.

## 4. Что готово (всё работает)

- Проекты: создание (с выбором цвета), архив/возврат, переименование, цвет, версия.
- **Удаление проекта:** только из архива. Подтверждение словом «DELETE».
- **Градиент-баннер проекта:** кнопка «🎨 Изменить фон» в шапке-баннере → пикер 12 пресетов.
  Градиент хранится в `projects.gradient`. Видно на карточке главного экрана и в баннере.
  Архивные карточки тоже показывают градиент (приглушённый через opacity).
- **Тёмная тема:** кнопка ☾/☀ в шапке главной. Класс `.pb.dark` переключает CSS-переменные.
  Состояние в `localStorage` (`pb-dark`).
- **Чекбокс «Выполнено» (Asana-style):** круглая кнопка на каждой задаче (доска и список).
  Отмечает задачу как закрытую (`closed:true`) независимо от статуса. По умолчанию при входе
  в проект фильтр `showClosed:false` скрывает такие задачи. Кнопка «✓ Выполненные (N)» в
  строке фильтров показывает их (зачёркнутые). В списке чекбокс в отдельной колонке с разделителем.
- **Дедлайн и исполнитель:** поля `dueDate` и `assignee` в TaskPanel. На карточках доски — бейджи
  «скоро» / «просрочено».
- **Bulk-действия:** кнопка «Выбрать», выделение нескольких задач, массовая смена статуса/
  приоритета, удаление. Плавающий BulkBar снизу.
- Задачи: доска (drag между колонками и внутри) и список (группировка, dnd, инлайн-редактирование).
- **Скриншоты:** lazy-загрузка (только при открытии панели), сжатие до ≤1920px JPEG,
  хранение base64 в Firestore. Спиннер во время загрузки, сообщение если файл не влез.
  Просмотр на весь экран, кнопка «Скачать». При удалении задачи файлы чистятся отложенно.
- **Realtime:** `onSnapshot` — изменения видны всем без перезагрузки.
- **Ctrl+Z / ⌘+Z (отмена):** откатывает создание/удаление/редактирование задач, статусов,
  проектов. История 40 шагов на вкладку. Тост «Отменено: …».
- **#N + ссылка на задачу:** «Скопировать ссылку» → `#task=<uuid>`. Переход по ссылке
  открывает задачу у любого.
- **Расширенная фильтрация:** платформа, приоритет, статус, версия, тег, номер #N, дата создания
  (пресеты + диапазон), showClosed. Счётчик активных фильтров.
- **Глобальный поиск:** строка на главной, ищет по всем проектам, открывает задачу напрямую.
- **Статистика:** вкладка открывается первой. Карточки (всего/открытые/на удержании/готово),
  разбивка по статусам/приоритету/платформе, двойной спарклайн «создано vs выполнено» за 30 дней.
  `isHoldStatus()` — ключевые слова для «заморозки» (hold/не фикс/не в билде и т.д.).
- **Теги:** 10 глобальных + кастомные на проект. Добавление/удаление в TaskPanel. Фильтр.
- **История изменений:** коллекция `activity`, анонимный лог, лениво при открытии задачи.
- **Мобильная оптимизация:** панель задачи на весь экран (≤768px).
- **PWA:** `public/manifest.json`, `theme-color`, `apple-mobile-web-app-capable`.

## 8. На что обратить внимание / нюансы

- **`closed` vs `completedAt`:** `closed` — ручная пометка «выполнено» (независимо от статуса).
  `completedAt` — автоматический timestamp при попадании задачи в последний статус. Оба поля
  независимы. Статистика использует `completedAt` для спарклайна.
- **`isHoldStatus(label)`** — дублируется в `StatsView.jsx` и `ProjectGrid.jsx`. Если нужно
  изменить ключевые слова заморозки, менять в обоих местах.
- **Пикер градиента** рендерится с `position:fixed` (вне карточки), т.к. `overflow:hidden` +
  `transform` на карточке обрезает `position:absolute` попапы.
- **Firebase вместо Supabase:** не путать. Supabase больше НЕ используется. Старые файлы
  `src/lib/supabase.js` и папка `supabase/` могут остаться в репо — они неактивны.
- **Firestore лимит 1 МБ на документ:** большие скриншоты (>~900 КБ base64) не сохранятся.
  При ошибке пользователь увидит красный текст под блоком скриншотов. Решение — уменьшить
  качество сжатия в `lib/image.js` (параметр `quality`, сейчас 0.85) или `maxDim` (1920).
- **Кэш скриншотов (`shotsCacheRef`):** `Map<taskId, {shots, loaded}>` в `useRef`.
  Живёт только в текущей вкладке, сбрасывается при перезагрузке. Защита от гонки:
  uploading-превью кладётся в кэш сразу, чтобы `rebuild()` от Firestore его не перетёр.
- **Ctrl+Z и скриншоты при удалении задачи.** Удаление `attachments` из Firestore —
  отложенное (finalize при выходе записи со дна стека отмены). Если перезагрузить вкладку
  до финализации — записи в Firestore останутся, но после повторного удаления очистятся.
- **Отмена — на текущей вкладке.** Стопка живёт в браузере, очищается при перезагрузке.
- **Включить вход** (когда понадобится): `AUTH_REQUIRED=true` в `useAuth.js` + настроить
  Firebase Authentication. Экран входа (`LoginScreen.jsx`) уже готов.
- **Деплой:** агент коммитит, пушит пользователь через GitHub Desktop.
