# Protoboard — бриф для сборки (этап 2)

Документ для передачи в **Claude Code**. Вставь содержимое раздела «Стартовый запрос» первым сообщением, и приложи файл `protoboard.jsx` как образец интерфейса.

## Что строим

Веб-трекер задач и багов для команды (~15–20 человек), по прототипам мобильных игр. Упрощённая Asana. Один проект = один прототип игры. Командный доступ по ссылке, изменения видны всем в реальном времени.

## Стек

- **Фронтенд:** React (за основу взять готовый прототип `protoboard.jsx` — он определяет внешний вид и UX).
- **Бэкенд / база / realtime / хранилище:** Supabase (Postgres + Realtime + Storage).
- **Хостинг:** Cloudflare Pages (деплой из GitHub-репозитория).
- Всё в рамках бесплатных тарифов.

## Функционал (зафиксирован на прототипе)

- Список проектов на главной; создание новых, архивация/возврат из архива.
- У проекта есть редактируемая **версия** (билд, напр. `v0.4`).
- Внутри проекта — два вида одних и тех же задач: **доска** (колонки `To Do → Ready to Check → Done`, drag-and-drop) и **список**.
- Фильтр по платформе внутри проекта: **Все / iOS / Android** (задачи с платформой «Общая» видны под обоими фильтрами).
- Карточка задачи: заголовок, статус, приоритет (Высокий/Средний/Низкий), платформа (iOS/Android/Общая), версия, описание, доп. условия/инфо, **скриншоты** (загрузка картинок).
- Версия задачи при создании подставляется автоматически из текущей версии проекта, далее правится вручную.
- **Авторизация пока НЕ нужна** — доступ по ссылке. Заложить так, чтобы добавить вход позже без переделки.
- **Realtime везде:** любое изменение у одного человека сразу видно остальным.

## Модель данных (ориентир)

**projects**
- `id` (uuid, pk)
- `name` (text)
- `build` (text, версия проекта, напр. "v0.1")
- `archived` (bool, default false)
- `created_at` (timestamptz)

**tasks**
- `id` (uuid, pk)
- `project_id` (uuid, fk → projects.id)
- `title` (text)
- `description` (text)
- `notes` (text — доп. условия/инфо)
- `priority` (text: 'high' | 'med' | 'low')
- `status` (text: 'todo' | 'check' | 'done')
- `platform` (text: 'ios' | 'android' | 'both')
- `version` (text — билд, в котором найдено; default = текущий build проекта)
- `sort_order` (int — порядок в колонке)
- `created_at` (timestamptz)

**attachments** (скриншоты)
- `id` (uuid, pk)
- `task_id` (uuid, fk → tasks.id)
- `file_path` (text — путь в Supabase Storage)
- `name` (text)
- `created_at` (timestamptz)
- файлы складывать в Supabase Storage bucket, напр. `screenshots`.

## Realtime и хранилище

- Подписки Supabase Realtime на таблицы `projects` и `tasks`, чтобы UI обновлялся вживую у всех.
- Скриншоты грузить в Storage, в задаче хранить ссылку/путь, а не сам файл.
